I just open-sourced 589 AI tools for managing 12 CMS platforms.

It's called CMS MCP Hub.

Here's the problem:
Every CMS has its own API. Its own auth. Its own quirks.
If you want AI to manage your WordPress, Shopify, or WooCommerce — you're writing custom integrations from scratch.

I built a better way.

CMS MCP Hub gives any AI assistant (Claude, Cursor, Windsurf, Copilot) full control over your CMS. One command. No plugins. No custom code.

You say: "Create a blog post, add a featured image, optimize SEO, and publish"
AI does it. End to end.

What's inside:

- WordPress — 169 tools (posts, pages, media, plugins, themes, menus, SEO, blocks, fonts, widgets)
- Shopify — 147 tools (products, orders, customers, inventory, collections, discounts)
- WooCommerce — 95 tools (products, orders, coupons, tax, shipping, reports, batch ops)
- Ghost — 24 tools
- Webflow — 21 tools
- Payload CMS — 21 tools
- Wix — 21 tools
- Framer — 20 tools
- Contentful — 20 tools
- Yoast SEO — 18 tools
- Strapi — 17 tools
- Sanity — 16 tools

Plus a REST API Gateway with auto-generated OpenAPI spec — so you can use these tools from Python, n8n, Make, LangChain, or any HTTP client.

Getting started takes 10 seconds:

npx @cmsmcp/wordpress

That's it. Your AI assistant can now manage your entire WordPress site.

Tech stack: TypeScript (strict), Zod validation, 462+ tests, MIT licensed.

This took mass of effort to build. 12 APIs. 12 auth methods. Full CRUD — not shallow wrappers.

If you're building with AI agents and manage any CMS — this saves you weeks of integration work.
